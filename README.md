# Intentional Development

**From Domain Understanding to Deployed Features Through Shared Intent, Behavior, and Verification**

This repository contains the open book project *Intentional Development*.

The book explores software development as a problem of preserving intent. It argues that many modern practices — Domain-Driven Design, Behavior-Driven Development, testing, DevOps, observability, AI-assisted development, and Agile V — can be understood as complementary responses to specification drift.

## Repository structure

```text
book/
  manuscript/        Published/open manuscript chapters
  assets/            Diagrams, figures, illustrations

book-development/
  book-vision.md     Stable book vision and promise
  domain-model.md    Conceptual model for the book
  book-outline.md    Working table of contents
  chapter-specs/     Chapter-level specifications
  traceability-matrix.md
  notes/             Loose notes and fragments
  decisions/         Book architecture decisions
```

## Working principle

This book is developed using the same philosophy it describes:

```text
Vision
↓
Domain model
↓
Book architecture
↓
Chapter specifications
↓
Draft chapters
↓
Review
↓
Publication
```
